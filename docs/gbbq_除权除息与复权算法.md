# gbbq 除权除息数据与复权算法说明(对齐通达信)

> 本文说明 tdx 项目中 `gbbq` 股本变迁数据的语义,以及复权(前复权/后复权)算法。
> **复权口径基准 = 通达信桌面客户端**,目标:复权数据与通达信桌面端逐分一致。
> 算法结论经独立 Python 工程项目逐点核对(通达信桌面端 + 东财官方分红交叉验证),

---

## 1. gbbq 数据语义

每条记录:`code + date + category + 4 个字段(C1..C4)`。字段语义**随 category 变化**:

| category | 含义 | C1 | C2 | C3 | C4 |
|---|---|---|---|---|---|
| **1** | **除权除息(XRXD)** | 分红(元/10股) | 配股价 | 送转股(股/10股) | 配股(股/10股) |
| 2/3/5/7/8/9/10 | 股本快照(Equity) | 盘前流通 | 前总股本 | 盘后流通 | 后总股本 |
| 6 | 增发新股 | _ | 发行价 | 增发股数 | _ |
| 11/12 | 扩缩股 | _ | _ | 缩股比例 | _ |
| 13/14 | 送认购/认沽权证 | 行权价 | _ | 数量 | _ |

> 解析见 `protocol/model_gbbq.go`。复权**只用 category=1(XRXD)**。
> 股本快照仅用于换手率(`Equity.Turnover`)与交叉校验,不参与复权。

---

## 2. 复权是【仿射变换】,不是【比例缩放】

通达信前复权(qfq)/后复权(hfq)都满足:
```
price_adj = A · price_raw + B
```
- `A` = 乘法因子(送股/转增/配股累积)
- `B` = 加法偏移(现金分红累积,并随后续送配缩放)

> ⚠️ 旧实现用**纯比例因子**(`factor = 除权后昨收 / 昨收`,逐日累乘)。该模型只能处理
> "纯送转",一旦有现金分红就会系统性偏离(实测命中率极低)。**必须改为仿射模型。**

### 2.1 逐 ex-day 系数(category=1 字段)
```
m = (10 + 送转股 + 配股) / 10
c = (分红 − 配股 × 配股价) / 10
```
- 标准除权参考价(即 `XRXD.Pre`):`P_adj = (P·10 − 分红 + 配股×配股价) / (10 + 送转 + 配股) = (P − c) / m`

### 2.2 前复权(QFQ):锚定最新交易日 A=1,B=0,自今向过去回溯
```
遇 ex-day:  A ← A / m
            B ← B − A_new · c     (A_new 为更新后的 A)
```

### 2.3 后复权(HFQ):锚定最早交易日
```
hfq = (qfq − B0) / A0      (A0,B0) = 最早交易日的 QFQ 系数
```

### 2.4 价格输出舍入:四舍五入(ROUND_HALF_UP)
通达信用四舍五入(逢五进一)到 2 位小数,**不是**银行家舍入。Go 实现用
`math.Round(x*100)`(`math.Round` 对 .5 远离零取整,正好等于逢五进一),
见 `roundHalfUpYuan`。复权价由 `Factor.QFQPrice(raw)` / `Factor.HFQPrice(raw)` 输出。

## 3. 【关键】停牌期 ex-day 复合事件:按"日期位置"施加,不按"交易日命中"

**Bug(旧实现 `XRXDs.Pre`)**:把 ex-day 与 kline **按日期精确匹配**,停牌日的 ex-day
找不到对应 kline,fallback 只给"第一个 ≥ ex-day 的 kline"施加**一个**事件(`break`)。
当**多个 ex-day 落在同一停牌缺口**(股改/大比例转增连续停牌),只有一个被施加,
其余丢失 → 该 ex-day 及之前所有历史日的复权因子被污染。

**正确做法**:自今向过去遍历交易日,对每个交易日 d,把所有 `exday > d` 且尚未施加的
事件**依次复合**进 (A,B)。两事件同缺口时自动合成 `m = m1·m2`、加法项链式缩放。

实证:
- 600519 茅台股改区(20060519 m=2.0 与 20060524 m=1.12,停牌至 20060525)→ 合成 m=2.24
- 000651 格力 @20150703(10送10派30,含停牌)→ 修复后与通达信一致

## 3bis. 【关键】未来 ex-day(已公告未除权)不得提前施加

gbbq 文件常含**已公告但 ex-day 在未来**的除权记录。前复权锚定**最新交易日** A=1,B=0,
若把 ex-day 晚于最新交易日的事件也施加进去,会让锚定日 B≠0,整条历史被平移一个**未发生**的
分红/送转额。

实证:600887 伊利 gbbq 含 `20260605 派9元`(在最新交易日之后),旧逻辑使
2016 至今前复权**全程偏低 0.90 元**(=9/10)。通达信/同花顺都只在除权日当天才生效。

**修复**:`PreKlines.Factors()` 收集事件时**丢弃 ex-day 严格晚于最新交易日的事件**。
回归测试 `TestFactors_FutureExDayIgnored`。

## 4. 已知无法对齐的特例(代码注释中举例)

### 4.1 600519 茅台 2006-05-26 开盘:孤立 0.005 边界点
- 本算法 qfq 开盘 = −260.97,通达信桌面端 = −260.98。
- 已排除 raw 来源(.day 与同花顺均为 39.49)、float 漂移(50 位 Decimal 重算 B 仍差 0.005)、
  fenhong 量化(均 <1e-4)。这是**真实 0.00505 差**。
- 任何"逐 ex-day 把因子 round 到 N 位"的模型能凑中此点,但会打坏已对齐的 2006-04-25
  收盘(−260.48 → −260.42)。
- 结论:通达信桌面端对该股改停牌簇边界日的**孤立内部处理差异**,无法用统一规则复现,
  否则牺牲其它已对齐点。**仅记录,不做硬编码补丁。**

### 4.2 600519 茅台 2006 股改前 / 000008 早期(1992-1993)
- 这两段是**同花顺**侧口径偏差(同花顺自身偏离官方送配),与通达信无关。
- 本算法对齐通达信,这两段与通达信一致,与同花顺差异属同花顺问题。

## 5. 实现位置

- 数据解析:`protocol/model_gbbq.go`(`gbbq.Decode`、`Gbbq.XRXD`、`Gbbq.Equity`)
- 复权算法:`protocol/model_gbbq.go`
  - `XRXD.mc()` 单事件仿射系数 (m, c)
  - `XRXDs.Pre(ks)` 把全部事件挂到各 `PreKline`(按日期位置复合的前置)
  - `PreKlines.Factors()` 输出每日仿射系数 `Factor{QFQMul,QFQAdd,HFQMul,HFQAdd}`
  - `Factor.QFQPrice(raw)` / `Factor.HFQPrice(raw)` 仿射 + 四舍五入得复权价
  - `roundHalfUpYuan` 逢五进一到 2 位小数
- 兼容字段:`Factor.QFQ/HFQ`(旧的纯比例因子)保留,仅纯送转无分红时与仿射一致,
  有现金分红时为近似,精确复权请用 `QFQPrice/HFQPrice`。
- 对外接口:`gbbq.go`(`IGbbq.GetFactors` → `GetXRXDs(code).Pre(ks).Factors()`),未改签名。
- 测试:`protocol/model_gbbq_test.go`(600519 股改复合 / 000651 配股+大送转 / 舍入边界 /
  已知特例 / HFQ 一致性),全部用通达信实盘值断言。
